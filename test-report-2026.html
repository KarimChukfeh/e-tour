⚙️ Technical Specifications

Core Protocol Features

  • Modular Architecture: 6 specialized modules (Core, Prizes, Matches, Escalation, Raffle, GameCache)
  • Delegatecall Pattern: Stateless modules execute in game contract context
  • Storage Optimization: Removed prizes array and Mode parameter (~200+ lines of code)
  • Simplified Prize Distribution: First place always gets 100% of prize pool
  • Match Cache: 1000-entry circular buffer for historical match data
  • Finals Preservation: Finals remain in live storage until next tournament starts

Economic Model

  • Entry Fee Split: 90% prize pool, 7.5% owner, 2.5% protocol
  • Protocol Raffle: Progressive thresholds with 10% reserve
  • Raffle Distribution: 20% owner, 80% random winner
  • Failed Prize Fallback: Accumulates to protocol raffle pool
  • Wei Precision: All prize calculations maintain exact wei accuracy

Anti-Stalling Mechanisms

  • Level 1: Normal timeout claim by opponent
  • Level 2: Advanced players can force eliminate after delay
  • Level 3: External players can claim stalled match slots
  • Enrollment Escalation: Solo players can reset window, external players can claim pool
  • Time Banks: Fischer increment system (15 seconds per move)

Security Features

  • ReentrancyGuard: All state-changing functions protected
  • CEI Pattern: Checks-Effects-Interactions order enforced
  • Failed Send Handling: Graceful fallback for prize distribution failures
  • Input Validation: Comprehensive bounds checking on all parameters
  • Randomness: Uses block.prevrandao for unpredictable outcomes
