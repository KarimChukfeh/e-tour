📍 Player Activity Tracking System (7 tests) - NEW FEATURE ✨ ▼
What is Player Activity Tracking?
A gas-efficient system that allows players to query their current tournament participation state without iterating through all tournaments. Players can instantly see which tournaments they're enrolled in (but haven't started) vs. which they're actively competing in.

Key Innovation: Automatic state transitions with zero storage bloat. When tournaments start, all players are moved atomically from "enrolling" to "active" lists. When players are eliminated or tournaments complete, tracking data is automatically cleaned up.
Storage Architecture
struct TournamentRef {
    uint8 tierId;        // 1 byte
    uint8 instanceId;    // 1 byte
}                         // Total: 2 bytes (gas optimized!)

// Track tournaments where player enrolled but not started
mapping(address => TournamentRef[]) playerEnrollingTournaments;

// Track tournaments where player is actively competing
mapping(address => TournamentRef[]) playerActiveTournaments;

// O(1) lookup/removal via index mappings
mapping(address => mapping(uint8 => mapping(uint8 => uint256))) playerEnrollingIndex;
mapping(address => mapping(uint8 => mapping(uint8 => uint256))) playerActiveIndex;

What is Match Escalation?
A 3-level system to prevent players from stalling tournaments. If a player doesn't move in time:
Level 1: Opponent claims timeout win
Level 2: Advanced players force eliminate both stalled players
Level 3: External players can replace stalled players and take their slot

Level 1 - Timeout Claim (60s)
Opponent can claim victory after move timeout
Level 2 - Force Elimination (+120s)
Advanced player eliminates both stalled players
Level 3 - Slot Replacement (+180s)
External player claims slot and advances
